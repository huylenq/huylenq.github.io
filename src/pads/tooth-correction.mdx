# Correcting Tooth Orientation in CBCT Imaging

In cone-beam CT imaging, teeth don't sit neatly aligned with the scan axes. The jaw curves, the tooth tilts, and the resulting cross-sections are skewed — making measurements unreliable.

The fix is a sequential 3-axis rotation correction: project the tooth onto a plane, find reference points on its convex hull, and rotate to align. Repeat for each axis.

The convex hull of the projection (outline) shows how the apparent shape deforms. Points **A** and **B** are the reference points used to compute the correction angle.

## Finding Reference Points

Three methods for identifying the orientation axis from the hull:

<RadioGroup name="method" options={["Farthest hull", "Restricted edge", "Bounding rect"]} />

**Farthest hull point** divides the hull into quadrants from the centroid and picks the farthest point in each, then takes the pair with maximum separation.

**Restricted edge hull point** applies separate D_X and D_Y thresholds per quadrant, keeping only points beyond 50% of both extents. From those, it picks the most axis-aligned point in each quadrant (smallest min(|dx|, |dy|)), then takes the pair with maximum separation.

**Minimum bounding rectangle** fits the smallest enclosing rectangle (rotating calipers) and derives orientation from its long axis.

Try switching methods above — notice how A and B shift, changing the computed angle.

## Step-by-Step Correction

The correction applies sequentially: Z-axis twice (Steps 1-1 and 1-2, using the X-Y projection), then X-axis (Y-Z), then Y-axis (X-Z). The double Z pass is deliberate — the first rotation changes the hull shape, and a second pass refines the residual. Each step uses the already-corrected points from prior steps.

## Try Different Teeth

Different tooth morphologies respond differently to each method. Incisors are elongated (bounding rect works well), molars are more square (quadrant methods may be better).
